## 追加依頼2: `data-pdf-no-break` の適用網羅確認

### 背景
コミット `f8aa46e` で `data-pdf-no-break` 属性によるブロック単位の改ページ制御を導入した。瑕疵担保責任セクションには適用済みの想定だが、**他の長文ブロック**（特に備考の各サブセクション）には未適用の可能性があり、別案件データで同じページ割れ問題が再発するリスクがある。

### 対応

#### Step 1: 現状の適用箇所を報告
以下の形式で、現在 `data-pdf-no-break` を付与している要素を一覧で教えてほしい。

```
- <Section: 瑕疵担保責任・アフターサービス>   components/pdf/XXX.tsx:123
- <Section: 追加工事・設計変更の取扱い>       components/pdf/XXX.tsx:145
- ...
```

#### Step 2: 以下のブロックに漏れなく適用
**PDF 2ページ目構成要素をすべて個別ブロックとして `data-pdf-no-break` 付与対象にする**:

1. ◆ 瑕疵担保責任・アフターサービス（見出し + 本文一塊）
2. ◆ 追加工事・設計変更の取扱い（見出し + 本文一塊）
3. 備考 / 【現場環境・諸条件】
4. 備考 / 【連絡体制】
5. 備考 / 【近隣対応】
6. 備考 / 【特記】
7. その他、見出し + 本文構造になっている全ブロック

各ブロックは「見出し単独ページ末尾・本文次ページ」という分離が起きないよう、見出しと本文のラッパdivに属性を付ける。

#### 注意
- **長大ブロックに付けるとブロックごと次ページへ送られ逆に余白が増える**ケースがある。1ブロックがページの7〜8割を超えそうなら、より細かい単位（例: 【特記】の各箇条書き※単位）で分割するか、`data-pdf-no-break` を付けない判断も可
- 判断基準はClaude Code側に委ねる。迷ったらコメントで記載

### コミットメッセージ案


複数ブロックへの追加がまとまるなら1コミットでOK。

### 検証
- 既存の検証用データ（髙﨑邸案件）で全ページプレビュー
- 備考の長いケース（箇条書き10個以上）でテストデータを作って2〜3ページに渡るPDFを出力し、どの見出しも本文と分離しないこと
- ページ途中で大きな空白（次ブロックが送られた結果）ができていないこと
